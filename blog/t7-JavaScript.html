Ruby Hashes vs. JavaScript Objects

What are the differences?

This past week was filled with JavaScript, a new language to me. It was fairly easy to work with but the thing I found to be the most confusing is that there are several items in JS that behave or look extremely similar to items in Ruby - but are not. One of those confusions was JavaScript Objects. They resemble Ruby Hashes, but aren't quite the same.

Objects in JavaScript are containers for named values. Such as:

var cat = {type: "domestic shorthair", color: "black and white", vaccinated: true};

The name:value pairs are called properties. The name (or key) is always a string. If you enter an integer as a key, JS will automatically convert it to a string. Methods are another named value that can be defined in an object. When methods are defined as part of an object, they are defined in properties as function definitions.

var cat = {type: "domestic shorthair", color: "black and white", vaccinated: true, ageHuman: 7, ageCat: function(){return cat.ageHuman * 7}};

Ruby Hashes act more like dictionaries - keys pointing to values. The keys in hashes can be any type - symbol, string, integer. Basically, anything that you want to point to a value can be used as a key. Hash pairs look similar to JavaScript name:value pairs.

Another difference is how items are accessed within a JS object vs. a Ruby hash.

JavaScript:
cat.type //returns "domestic shorthair", type is a string

Ruby:
cat["type"] # returns "domestic shorthair", where type is a string
cat[:type] # example where type is a symbol

Ruby hashes also have built-in methods, but JavaScript objects do not.

As my significant other recently advised, when learning a new language, take absolutely nothing for granted. Even though it may look alike and behave in a similar manner, there are nuances that we have to become familiar with and discover so that we can properly proceed with our code. JavaScript objects may look like Ruby hashes, but they honestly behave more like Ruby objects.
