Arrays & Hashes

Oh, the possibilities!

There are a lot of possibilities with arrays and hashes. They both act as indexed collections of information. And they can contain each other - for example, an array can contain hashes as values and a hash can contain arrays.

A list of values resides in an array. Each item on the list can be pointed to by using an index, which uses zero-based numbering. So, if you had the following array:

pets = ['cat', 'dog', 'fish', 'ferret', 'hamster']

If you asked for the first item in the array, you'd do it like this: pets[0] and the answer would be 'cat', since that is the value in the first (0) spot in the array. pets[1] would give the answer of 'dog'.

On the other hand, hashes (which are essentially a type of array) are a list of keys and values. Instead of using indexes to point to values, a hash uses keys.

pets {'cat' => 'Charlie', 'dog' => 'Chance', 'fish' => 'Candy', 'hamster' => 'Wendell'}

In this instance, if you asked for the first item in the array, you'd be asking for the key & value. If you just wanted the value, you'd ask like this: pets['cat'] and the answer whould be 'Charlie'. If you wanted to know the value associated with 'hamster', you'd type pets[hamster] and receive 'Wendell' as the value.

Now, why do hashes and arrays open so many possibilities? Because of what you can do with them. Let's pretend that you're moving and you want to send out your new address to everyone that you know. You can use a hash to iterate over each person and their contact information to quickly print out envelopes for your announcement! Let's say that you're doing inventory at a store and have a huge list of items and the count in stock. You can use a hash to sort the items in anyway that you wish and not risk the counts being lost.

puts pets.length #4, there are four sets of keys and values in the hash named pets
[6, 'jun'] => "June" Use an array as a key, so either 6 or 'jun' will point to June.
pets.has_value?('Candy') #true

You can use methods to find if an item exists, to add items to your list, to delete items, to calculate, divide, change values, etc... See? Possibilities.

And, you can have arrays within arrays. Basically, it's like having a library filled with dictionaries, or a drawer containing multiple address books, or a folder containing different types of lists.

library = [frenchdict{}, spanishdict{}, websterdict{}, oxforddict{}]

As similar as hashes and arrays are to each other, I believe that hashes have more capabilities because they can use any object as the index.
